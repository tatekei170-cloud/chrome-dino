# 不確定要素・近似のまとめ

このプロジェクトで **本家 `chrome://dino` や Chromium ソースと 1:1 対応していない／検証していない** 点を一覧にする。  
数値は「調整用」であり、**仕様として確定したものではない**。

---

## 1. 時間・フレーム（再現性）

| 項目 | 内容 |
|------|------|
| **ループ** | `requestAnimationFrame` + **`step = 経過ms / (1000/60)`** で移動・加速・重力・雲・スポーン消費をスケール済み（実装済み）。 |
| **実時間** | 同じ `BASE_SPEED` でも **おおむね** Hz に依存しにくい。タブ非表示時は `dt` を 100ms でキャップ（長時間放置後の飛び防止）。 |
| **走りアニメ** | `rexAnimT`＋`REX_ANIM_RATE`。**本家**は `Trex.animFrames.RUNNING.msPerFrame = 1000/12`（ms）で 2 コマ切替。当コードは `REX_ANIM_RATE = 1` でそれに相当する意図（`step` で実時間寄り）。 |
| **その他** | `flashTimer`・ゲームオーバー UI は **1 rAF あたり** のまま。 |

---

## 2. 速度・スコア（独自定数）

| 定数 | 役割 | 不確定さ |
|------|------|----------|
| `BASE_SPEED` / `MAX_SPEED` / `ACCELERATION` | 障害物の横移動（論理 px / ループ）と加速 | 本家の `SPEED` 系とは別物。好みで変える前提。 |
| `CLOUD_PARALLAX` / `CLOUD_IDLE_PX` | 雲の移動 | 視覚用の割合。本家の `BG_CLOUD_SPEED` 式とは未照合。 |
| `SCORE_RATE` | `score += speed * SCORE_RATE` | 本家の距離メーター換算とは別。桁の伸び方は一致しない。 |

---

## 3. 論理解像度と表示（CSS）

| 項目 | 内容 |
|------|------|
| **キャンバス** | 論理 `W=800`, `H=250`（本家の 600×150 とは異なる）。 |
| **表示幅** | `.game-shell { max-width: 600px }` などで CSS スケール。論理 1px と画面 1px は一定比とは限らない。 |
| **レターボックス** | `.canvas-shell` の `aspect-ratio: 800 / 290` はレイアウト用。本家の余白とは無関係。 |
| **devicePixelRatio** | キャンバスを HiDPI 用に `width/height` 属性と CSS を二重に合わせる処理はしていない（ぼやけ／太さの差が出うる）。 |

---

## 4. 当たり判定

| 項目 | 内容 |
|------|------|
| **プレイヤー（画像あり）** | PNG の `naturalWidth/Height` から **割合 inset**（左 12%・右 34% など）で矩形を縮めている。透明余白の実形状とは一致しない可能性あり。 |
| **しゃがみ** | inset を別割合にしているが、Player 04 の実画に対して未計測。 |
| **障害物** | 描画は `drawImage(..., o.w, o.h)` で伸縮。当たりは `o.x+2` 等の矩形。スプライト内の余白とズレうる。 |
| **翼竜** | 図形フォールバック時と画像時で精度が変わる。 |

---

## 5. 障害物の出現間隔

| 項目 | 内容 |
|------|------|
| **実装** | `nextSpawn -= speed` と固定レンジ `280 + random*220` などの簡易モデル。 |
| **本家** | 障害物幅・`speed`・`GAP_COEFFICIENT` 等から **動的に gap を計算**する方式（このコードにはない）。 |
| **結果** | 詰まり方・難易度カーブは本家と一致しない。 |

---

## 6. ジャンプ・重力

| 項目 | 内容 |
|------|------|
| `GRAVITY` / `JUMP_V` / `INTRO_JUMP_V` | 毎ループ加算の単純モデル。 |
| **本家** | 速度に応じた跳ね方・`INITIAL_JUMP_VELOCITY`・`DROP_VELOCITY` など別ロジック。 |
| **結果** | 速度が変わっても `JUMP_V` は固定（本家ほど速度連動しない）。 |

---

## 7. イントロ（開始ホップ）

| 項目 | 内容 |
|------|------|
| **終了条件** | 「一度空中に出たあと着地」で本走開始。細かいタイミングは本家の `playingIntro` 等とは未照合。 |
| **`INTRO_JUMP_V`** | 手調整。本家のイントロ移動量とは別。 |

---

## 8. ランダム・ゲームルールの端数

| 項目 | 内容 |
|------|------|
| サボテン／翼竜の **出現率・サイズ** | `Math.random()` と閾値（例: 0.18, 0.45）。本家のテーブルとは未照合。 |
| **昼夜** | `nextNightScore`（例: 700 刻み）と `night` トグル。本家のタイミングとは未照合。 |
| **地面の細線** | `dist * 0.5` や `% 20` など見た目用。本家ホライズン線ロジックとは別。 |

---

## 9. UI・素材まわり

| 項目 | 内容 |
|------|------|
| `GO_CHROME_ICON_MS` | ゲームオーバー後、リトライアイコン切替の待ち時間（800ms）。本家値の検証なし。 |
| **タイトル** | HTML 画像＋キャンバス切替。本家の同一キャンバス構成とは異なる。 |
| **フォント** | スコア HUD のフォント・サイズは雰囲気用。 |

---

## 10. 未実装・省略（比較対象としての不確定）

- モバイル向け速度係数（本家に相当するもの）
- サウンド
- 本家の `deltaTime` ベースの更新
- ゲームパッド（本家にある処理の有無・仕様はこのプロジェクト外）

## 使い方

- **「本家と同じか？」** と聞かれたら、このファイルの該当箇所は **No または未検証** と答えられる。
- 数値を変えるときは **`js/game.js` 上部の定数** と、このリストをセットで見ると、何が「好み調整」か分かりやすい。
