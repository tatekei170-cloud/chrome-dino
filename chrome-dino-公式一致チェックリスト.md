# Chrome オフライン恐竜ゲーム（`chrome://dino/`）との完全一致に向けた要素一覧

目的が「公式と完全一致」なら、以下をすべて本家の挙動・見た目・データに合わせて詰める必要があります。  
（本家は Chromium のソースとスプライトシートに実装されている。著作権のあるアセットの**無断コピー**は避け、**自作で同ピクセル・同ロジック**に寄せる前提で整理しています。）

---

## 1. 表示・解像度・スケーリング

- [ ] 論理キャンバスサイズ（本家は幅 600px 相当のランナー領域＋高さはスプライトに準拠）
- [ ] HiDPI / `devicePixelRatio` 対応（ぼやけ防止とクリック座標）
- [ ] フルスクリーンやウィンドウリサイズ時のスケール（本家は埋め込みコンテキスト依存）
- [ ] `image-rendering` 相当のピクセル表示（拡大時もドットが潰れない）

---

## 2. スプライト・アニメーション（ビット単位）

- [ ] 恐竜：待機・走り 2 コマ・ジャンプ・しゃがみ 2 コマ・死亡コマ
- [ ] サボテン：単体・複合など**全バリエーション**のフレームと当たり矩形
- [ ] 翼竜（プテラ）：高さ違い × 羽ばたきコマ
- [ ] 地面・遠景・雲：スクロール速度差（パララックス）とループ周期
- [ ] 星（夜モード用）・月 など夜用レイヤーがあるなら同様に
- [ ] ゲームオーバー：「GAME OVER」文字、再起動ボタン、Chrome ロゴ→リロードアイコンの**タイミングとフレーム**
- [ ] スコア用数字スプライト（本家は画像数字の可能性が高い。等幅フォント描画との差）

---

## 3. 昼夜（ナイトモード）

- [ ] 切り替え条件（本家は**スコア閾値**でトグルし、以降も一定間隔で反転する仕様）
- [ ] 切り替え時の**色反転／パレット差し替え**対象（背景・スプライト・地面・雲など）
- [ ] 夜用スプライトが別テクスチャか同一テクスチャの反転かをソースに合わせる

---

## 4. 物理・入力

- [ ] 重力・初速・最大落下速度・接地判定の y（足元アンカー）
- [ ] ジャンプ：キー連打・長押しの扱い（本家は 1 回ジャンプのみ等）
- [ ] しゃがみ：`ArrowDown` のみ有効な区間（空中では無効など）
- [ ] スタート：Space / タップの**初回のみ**タイトル→プレイ遷移
- [ ] ゲームオーバー後：Space で**リセット→タイトル相当→再スタート**の遷移順

---

## 5. 速度・難易度曲線

- [ ] 初期速度、最大速度、**スコアまたは経過時間に対する加速式**（係数はソースの定数と一致）
- [ ] 障害物生成間隔の**距離ベース／時間ベース**のロジック一致
- [ ] 翼竜が出現し始めるスコア閾値
- [ ] 各障害物タイプの**重み付きランダム**（出現率テーブル）

---

## 6. 当たり判定

- [ ] 恐竜・各障害物の**AABB または本家が使っている矩形**（スプライトごとに複数矩形の可能性）
- [ ] しゃがみ時の当たり箱の縮小
- [ ] 無敵時間・衝突後のフレーム停止の有無

---

## 7. スコア・ハイスコア

- [ ] スコア増分のタイミング（距離換算か固定ティックか）
- [ ] 表示桁・ゼロ埋め・「HI」と現在スコアの並び
- [ ] ハイスコアの保存先（本家は `localStorage` のキー名・形式が決まっている）
- [ ] 一定スコアで**プレイ速度が上がる以外のイベント**（昼夜など）との連動

---

## 8. 音（本家にある場合すべて）

- [ ] ジャンプ音
- [ ] スコアマイルストーン音（100 の倍数など、要ソース確認）
- [ ] ゲームオーバー音
- [ ] 再生タイミング・ボリューム・ループなし等

---

## 9. UI テキスト・ロケール

- [ ] オフライン画面の「Space で開始」系文言（言語・フォント）
- [ ] ゲームオーバー周りの案内（本家は英語 UI が基本。日本語はブラウザ/OS 依存の別画面の可能性あり——**再現対象を `chrome://dino` 固定にするか**要定義）

---

## 10. 状態機械（全体フロー）

- [ ] `INTRO` / `PLAYING` / `CRASHED` / `PAUSED` 等の公式ステートに相当する遷移
- [ ] クラッシュ直後の**入力無効フレーム**の有無
- [ ] バックグラウンドタブ・フォーカス外での停止（本家ブラウザ挙動との差は「ゲーム単体」では省略可と明記）

---

## 11. 実装ソースの参照先（調査用）

完全一致を目指すなら、次を**一次情報**として読むのが最短です。

- Chromium の **Runner / T-Rex** 関連の JavaScript（オフライン画面用）
- 同梱の **スプライト 1 枚（atlas）** と **座標定数**
- 上記に対応する **単体テストやコメント**（あれば）

（リポジトリ内の正確なパスは Chromium のバージョンで変わるため、検索キーワードは `runner`, `t-rex`, `offline` など。）

---

## 12. 検証方法

- [ ] 本家と**同じ入力列**（ジャンプタイミング）で生存時間・スコアを比較
- [ ] スクリーン録画をフレーム比較（昼夜切替・障害物パターンの再現性）
- [ ] 定数をソースから**コピペではなく**自作定数で一致させる（著作権配慮）

---

## 13. いまの自作版から見た「大きなギャップ」になりやすい所（要約）

| 領域 | 完全一致に必要になりやすいこと |
|------|-------------------------------|
| グラフィック | atlas 単位のフレーム・色・夜パレット |
| ゲームオーバー | ピクセルフォント・ボタン・アイコン切替フレーム |
| ロジック | 速度式・障害物テーブル・生成間隔 |
| 当たり判定 | 矩形の数値がソースと一致しているか |
| 音 | 有無・タイミング・ファイル形式 |
| ナイトモード | 存在自体と切替条件・見た目 |

---

## 14. プロジェクトで決めておくとよい「完全一致の定義」

- 「**見た目と操作とスコア曲線**まで」なのか  
- 「**ソース定数と同一の数値**まで」なのか  
- 「**音まで**」なのか  

をレポートに書いておくと、採点者・自分の両方にとってブレません。
